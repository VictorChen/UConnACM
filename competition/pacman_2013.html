How to play:

PacMan is an old arcade game developed by Namco. If you don't know how to play PacMan then you don't have a childhood. But, don't worry, it's simple. You control pacman using your arrow keys (up, down, left, right) and your goal is to collect all the coins on the screen. However, you must do so while avoiding "ghosts". There will be four ghosts trying to hunt down pacman. If a ghost touches pacman then he will die. There are also powerups located in different places that pacman can pick up. When pacman picks up a powerup, he will be able to kill the ghosts for a short duration by touching them. Once all of the coins are collected, you win.

Programming competition:

ACM's PacMan competition is based on the traditional PacMan game, but with a few changes. There will be two pacmans in the game at once. Each pacman is controlled by a different person with JAVA code instead of arrow keys. There are no powerups in the game and pacmans can bypass each other. Once both pacmans have died, the pacman with the most coins collected will be the winner.

Information provided to each player:

  • Location of ghosts
  • Location of both pacmans
  • Map of the game (walls, coins)
  • Whether or not a pacman is currently dead

Getting started:

Open up the file PacMan2.java

You should notice 3 things about the code:

  1. Long comment
  2. empty init() method
  3. move() method

The long comment that you see is a detailed documentation of the available methods that you are allowed to use. You should read this!

The empty init() method is similar to a constructor. This method gets called when pacman is initially created (start of a round). You should initilize any variables that you have here.

The move method is the most important part of the program. This is where you will write your algorithm to control the pacman. This method returns an integer: 1 = go up, 2 = go down, 3 = go left, 4 = go right and 5 = continue going the current direction. If you want your code to be more readable, you can return Constant.UP, Constant.DOWN, Constant.LEFT, Constant.RIGHT or Constant.CONTINUE instead of numbers.

Rules:

  1. You must submit 2 versions of your code: PacMan1.java and PacMan2.java.
  2. You can ONLY use methods that you are allowed to use (read the comment in PacMan1.java or PacMan2.java for the list of methods).
  3. You should ONLY write code in PacMan1.java and PacMan2.java. Do not touch any other files.
  4. Inside PacMan1.java or PacMan2.java, feel free to add as many private variables or methods as you'd like
  5. Do NOT edit the last 2 lines of code inside PacMan1.java and PacMan2.java.
